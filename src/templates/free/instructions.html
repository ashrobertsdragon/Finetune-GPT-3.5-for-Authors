How to Finetune GPT 3.5 using
ProsePal Finetuner

1. Sign Up for OpenAI

If you haven't already, the first thing you'll need to do is create an OpenAI account by going to OpenAI's signup page and follow the prompts to register.

2. Acquire Your API Key

If you haven't already, generate your OpenAI API key. This is a code that identifies you when using OpenAI's API. Visit the API keys page, click on "Create new secret key", and make sure to store this key in a safe placeâ€”you won't be able to retrieve it from the site again for security reasons.

Note: ProsePal does not save your key either. As soon as finetuning is complete, the key is deleted from memory so you won't have to worry about it being stolen.

3. Prepare Your Manuscripts

Your books need to be in text format, with chapters separated by three asterisks (***). If your texts aren't formatted this way, we can help with that. Click on "Convert your ebooks here" to get started.

4. Upload Your Texts

Once your files are ready, use the "Choose files" button to upload them.

5. Enter a System Message

In the system message box, you'll need to provide a description of your writing style. This might look something like, "You are a junior writer who writes in the style of [Author Name], known for [describe the style]." If you need a bit of help, our ProsePal's Writing Style Describer is available for ChatGPT Plus (the paid version) subscribers.

6. Choose Your Fine-Tuning Method

You'll see a few options for how the finetuner should split your text, such as "sliding window" at chapter or book level, or "Dialogue/Prose." These options will create a fine tuned model that understands slightly different things.

  • The "sliding window" methods focus on small sections at a time. It takes one piece, called a chunk, as the input and then the next as the response. But for the next set, the response chunk becomes the input chunk, and the chunk after that is the new response. And so on down the line. The chapter level does this within each chapter, while the book level option does this with the chapters themselves.

    Warning, this method causes your fine tuning instructions to be twice as long, which doubles the cost of fine tuning.

  • The "Dialogue/Prose" option splits each paragraph between the dialogue and narrative sections and tells GPT 3.5 to write as many sentences as there were of each kind in the paragraph.

  • There's also an experimental "generate beats" option that will read the book files you upload and attempt to create scene beats for each chapter, and then direct the fine tuned model to write the chapter based on those beats, with the correct response being the orginal text.

    This also winds up having a little extra cost, but uses the base GPT 3.5 model which is pretty inexpensive these days, and will wind up being around 4 to 5 cents (in US dollars) per hundred thousand words uploaded. It will also take a bit longer to process, but again, it's pretty fast for AI.

7. Confirm Rights and Agree to Terms

You have to confirm that you hold the rights to the content you're uploading and to agree to the terms and conditions. If you want to teach the AI to write like somebody else, you are looking for a different tool. The whole point is to train the AI to sound like you, so why would you upload someone elses books?

8. Begin Fine-Tuning

Finally, hit the "Fine tune GPT 3.5" button. You'll see updates on the screen about the fine-tuning process, and once it's completed, you'll receive a message along with the name of your fine-tuned fodel which will start with "ft:gpt-3.5-turbo-0613".

A file will also automatically download. This file, formatted in something called JSONL or JSON lines, contains all the instructions that were used for finetuning GPT 3.5 and can be used for further fine-tuning or modifications in the future.

Your fine-tuned AI model will then be listed on your OpenAI account and can be used in various anywhere you can use your own versions of GPT 3.5 like OpenAI Playground, Rexy, or Novelcrafter.
